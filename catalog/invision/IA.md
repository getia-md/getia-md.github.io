# Information Architecture вЂ” InVision

## 1. Overview
InVision is a digital product design platform that pioneered **clickable prototyping** from static screens and later expanded into collaborative whiteboarding (Freehand) and design system management (DSM). While the company has scaled down its product suite, the IA reflects its historical role as a design collaboration hub вЂ” connecting designers, developers, and stakeholders through prototypes, design specs, and real-time collaboration boards. Key products include prototyping, Freehand, and DSM.

## 3. Navigation Model

| Level | Type | Details |
|-------|------|---------|
| invisionapp.com | Top nav | Logo, Product, Solutions, Pricing, Templates, Resources, Sign in, Sign up |
| Product | Dropdown | Freehand, Prototyping, DSM, Inspect, Integrations |
| Dashboard | Left sidebar | All, Spaces, Recent, Archived, Trash, Freehand, DSM |
| Prototype | Mode switcher | Build, Preview, Comment, Inspect, History |
| Freehand | Floating toolbar | Drawing tools, shapes, text, sticky notes, connectors |

**Key pattern**: InVision uses a mode-switching paradigm in prototypes вЂ” Build (add interactions), Preview (test), Comment (feedback), Inspect (handoff) вЂ” each mode shows the same screens with different overlays.

## 5. User Flows

### 5a. Create a prototype
1. New Project в†’ upload screen designs (PNG/JPG from Sketch, Figma, etc.)
2. Enter Build mode в†’ draw hotspots on interactive elements
3. Link hotspots to target screens в†’ set transition animation
4. Preview mode в†’ click through prototype on device frame
5. Share link в†’ stakeholders comment directly on screens

### 5b. Freehand brainstorming session
1. Create new Freehand в†’ choose template (or blank)
2. Share link with team в†’ real-time cursors visible
3. Add sticky notes, drawings, diagrams, wireframes
4. Vote on ideas (dot voting) в†’ use timer for timeboxing
5. Export as PDF or continue iterating

### 5c. Design system management (DSM)
1. Set up DSM в†’ add components from design tool
2. Add design tokens (colors, fonts, spacing)
3. Write documentation for each component
4. Add code snippets (HTML/CSS, React, Swift)
5. Team references DSM as single source of truth

### 5d. Developer handoff (Inspect)
1. Developer opens shared prototype в†’ Inspect mode
2. Click any element в†’ view specs (dimensions, spacing, colors, fonts)
3. Copy generated code (CSS, Swift, Android)
4. Download assets (images, icons in multiple resolutions)
5. Leave comments for questions в†’ designer responds
